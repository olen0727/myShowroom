# 專案開發規則

## 程式碼定位與修改

### 必須遵守的規則

1. **禁止依賴記憶中的行號**
   - Windows 使用 CRLF (`\r\n`) 換行符號，會導致行號計算錯誤
   - 每次修改前必須先用 `view_file` 或 `grep_search` 確認當前內容

2. **提供修改指引的正確方式**
   - ✅ 好的方式：「找到 `.skillCard:hover` 這段 CSS，修改其中的 `transform` 屬性」
   - ❌ 壞的方式：「在第 150 行修改」

3. **修改程式碼前的檢查清單**
   - [ ] 使用 `grep_search` 搜尋目標代碼片段
   - [ ] 使用 `view_file` 確認當前內容和行號
   - [ ] 確認 `TargetContent` 與文件內容完全一致（包含空格和縮排）

4. **文件編輯工具使用規範**
   - 如果連續編輯失敗 2 次，改為告訴使用者如何手動修改
   - 提供完整的「搜尋 → 替換」指引，而不是只說行號

## 溝通方式

- 當需要使用者修改程式碼時，提供「搜尋關鍵字」和「完整的修改內容」
- 避免只提供行號，因為行號可能不準確
