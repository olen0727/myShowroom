# 專案開發規則

## 程式碼定位與修改

### 必須遵守的規則

1. **編輯失敗時檢查**
   - 如果編輯失敗，使用 `view_file` 或 `grep_search` 檢查文件當前狀態
   - 找出失敗原因後再重試

2. **提供修改指引的正確方式**
   - ✅ 好的方式：「找到 `.skillCard:hover` 這段 CSS，修改其中的 `transform` 屬性」
   - ❌ 壞的方式：「在第 150 行修改」

3. **連續失敗的處理**
   - 如果連續編輯失敗 2 次，改為告訴使用者如何手動修改
   - 提供完整的「搜尋 → 替換」指引

## 換行符號設定

- 本專案使用 **LF** (`\n`) 換行符號
- Git 設定：`core.autocrlf = false`
- 如果遇到編輯工具匹配失敗，檢查是否有 CRLF 混入

## 溝通方式

- 提供「搜尋關鍵字」和「完整的修改內容」
- 使用 CSS 類名、函數名等具體標識，而非行號
